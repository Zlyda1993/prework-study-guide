HTML

  • HTML stands for Hypertext Markup Language.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage.
  • HTML is considered the frame work of the website.
  • An HTML attribute defines additional information about an element.
  • HTML attributes are always defined in the beginning element, never the closing element.
  • Elements that are nested inside other elements are called child elements.
  • An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.
  • Headings are exactly what they sound like: larger or more prominent titles and subtitles to organize a page.

CSS

  • Cascading Style Sheet (CSS) is a programming language used by developers to define how a webpages content is presented to users.
  • CSS defines the colors, aestetics, visual layout, and organization.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • External style sheet allows developers to keep all their CSS rules in a seperate file, which makes design changes easier.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.

Git

  • The Git command must precede every git command.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git checkout main: switches back to main branch off of a feature branch.
  • The -b: flag creates a new branch.
  • The feature/starter-code: is the name of the new branch. We have added the prefix, feature, to indicate that it is a feature.
  • It is a good practice to name the branches for the feature that will be developed by them to help indicate the purpose of the branch.
  • pwd (print working directory): displays the current working directory.
  • ls (list): displays the list of files and folders in the current directory.
  • cd (change directories): navigates between directories in the path.
  • cd ..: navigates backwards, one level at a time, through the path of your directory.
  • mkdir (make directory): creates a new directory.
  • touch: creates a file inside a directory.
  • start: opens a file in windows.
  • open: opens a file in apple.
  • ls -al ~/.ssh: to check for SSH key.
  • ssh-keygen -t ed25519 -C "your_email@example.com": to create a new SSH key.
  • git revent (insert sha): to revert to a previous version of a code(sha) type.
  • git add -A: lets git know to stage all files.
  • git commit -m "commit message": confirms the changes made and allows you to include a message to explain what changes where made to keep better track of what is being done and for incase anyone else is working on it too, they can understand what the change was.
